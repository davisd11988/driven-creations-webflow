3. HOW TO EDIT CONTENT
================================================================================

All content is in plain HTML files. To change text, images, or links:

CHANGING TEXT
  1. Open the relevant HTML file (e.g., index.html for homepage)
  2. Find the text you want to change
  3. Edit the text between the HTML tags
  4. Save the file
  5. Deploy (see Section 7)

  Example - Change the homepage hero subtitle:
    Find:    <div class="hero-subtitle">We build brands that break through</div>
    Change:  <div class="hero-subtitle">Your new tagline here</div>

CHANGING IMAGES
  1. Add your new image to the /images/ folder (or /images/work/ for projects)
  2. Open the HTML file
  3. Find the <img> tag and update the src="" path
  4. Save and deploy

  Example:
    Find:    <img src="images/work/forgent-thumbnail.jpg" alt="Forgent Power">
    Change:  <img src="images/work/new-image.jpg" alt="New Project">

CHANGING LINKS
  Find the <a> tag and update the href="" value.

  Example - Update a "View Live Site" link on a project page:
    Find:    <a href="https://old-url.com" target="_blank">
    Change:  <a href="https://new-url.com" target="_blank">

CHANGING THE VIDEO MODAL (Watch Episode 1 button)
  File:     js/shared.js
  Line:     ~24
  Current:  iframe.setAttribute('src', 'https://www.youtube.com/embed/im-qtaIxbYk?autoplay=1&rel=0');

  To change the video, replace "im-qtaIxbYk" with a new YouTube video ID.
  The video ID is the part after "youtu.be/" or "v=" in a YouTube URL.

================================================================================

STEP 1: DUPLICATE AN EXISTING PROJECT FILE
  Copy:   work/forgent-power.html

  a) <title> tag - Page title for browser tab
  b) Project Hero - Title, tagline, category tag, gradient colors
  c) Project Overview - Challenge text, Client, Services, Industry, Year
  d) View Live Site link - Update the href URL
  e) Showcase - Replace video/image source
  f) The Approach - Two blocks with title, description, image
  g) Results - Three stat cards (number + label)
  h) More Projects - Three related project cards at the bottom

STEP 3: ADD IMAGES/VIDEOS
  Add project images to:  images/work/
  Add project videos to:  videos/

STEP 4: ADD TO PORTFOLIO GRIDS
  Add a card linking to the new project in:
  - index.html (homepage portfolio section, ~6 cards shown)
  - work.html (full portfolio grid, all projects shown)

STEP 5: DEPLOY
  See Section 7.

7. HOW TO DEPLOY CHANGES (PUSH LIVE)
================================================================================

After editing files locally, you need to do TWO things:

--- STEP 1: SAVE TO GITHUB ---

Open Terminal. Run these commands:

  cd ~/Desktop/driven-creations-webflow
  git add -A
  git commit -m "Describe what you changed"
  git push

This saves your changes to the GitHub backup. But it does NOT update
the live site (Vercel is not auto-connected to GitHub).

--- STEP 2: DEPLOY TO VERCEL ---

In the same Terminal, run:

  npx vercel --prod --yes --scope derricks-projects-85a0451e

Wait for it to finish (usually 10-20 seconds). It will show:
  "Production: https://driven-creations-webflow-xxxxx.vercel.app"

Your changes are now live at https://drivencreations.com

--- QUICK ONE-LINER (do both at once) ---

  cd ~/Desktop/driven-creations-webflow && git add -A && git commit -m "Update site" && git push && npx vercel --prod --yes --scope derricks-projects-85a0451e

--- OPTIONAL: CONNECT GITHUB FOR AUTO-DEPLOY ---

If you want pushes to GitHub to automatically deploy:
  1. Go to: https://github.com/apps/vercel/installations/new
  2. Install the Vercel GitHub App on your account
  3. Grant access to the driven-creations-webflow repo
  4. Go to Vercel Dashboard > Project Settings > Git
  5. Connect the repository

After this, every "git push" will auto-deploy. No manual Vercel step needed.

9. USING CLAUDE CODE TO EDIT
================================================================================

You can use Claude Code to make any changes to the site. Here's how:

STARTING A SESSION
  1. Open Terminal
  2. Run: claude
  3. You're now in Claude Code

COMMON REQUESTS YOU CAN MAKE:

  "Change the hero title on the homepage to [new text]"
  "Update the YouTube video in the Watch Episode 1 modal to [URL]"
  "Add a new project page for [project name]"
  "Change the purple accent color to [new color]"
  "Update the testimonial from [client] to say [new quote]"
  "Add a new image to the about page"
  "Change the contact email address to [new email]"
  "Deploy the site to Vercel"
  "Push changes to GitHub"

Claude Code has access to all your project files and can:
  - Edit HTML, CSS, and JavaScript files
  - Add new pages
  - Run git commands (commit, push)
  - Deploy to Vercel via CLI
  - Check if the site is working

TIPS:
  - Be specific about what page and section you want to edit
  - You can paste URLs or text and ask Claude to update content
  - After Claude makes changes, ask it to deploy
  - Claude can also make multiple changes in one session

11. TROUBLESHOOTING
================================================================================

SITE NOT UPDATING AFTER DEPLOY
  - Make sure you ran the Vercel deploy command (Section 7, Step 2)
  - Clear your browser cache (Cmd+Shift+R on Mac)
  - Vercel caches images/videos for 30 days; use a new filename if needed

VERCEL CLI NOT WORKING
  - Make sure you're in the project folder: cd ~/Desktop/driven-creations-webflow
  - If "npx vercel" hangs, try: npx vercel login (re-authenticate)
  - If scope error appears, add: --scope derricks-projects-85a0451e

GIT PUSH FAILING
  - Check if you're logged into GitHub: gh auth status
  - If not: gh auth login
  - If merge conflicts: ask Claude Code to help resolve them

SSL CERTIFICATE ISSUES
  - Certificates auto-renew on Vercel
  - If issues arise: npx vercel certs issue drivencreations.com --scope derricks-projects-85a0451e

DNS NOT WORKING
  - Changes can take up to 48 hours to propagate (usually minutes)
  - Check current DNS: dig drivencreations.com A +short
  - Should return: 76.76.21.21

BROKEN IMAGES/VIDEOS
  - Check the file path in the HTML matches the actual file location
  - File names are case-sensitive
  - Make sure new files were committed (git add -A) and deployed

PAGE RETURNING 404
  - Vercel uses clean URLs (no .html extension needed)
  - /about works, /about.html also works
  - Check the file exists in the project folder
  - Check vercel.json for any redirect rules
